CHANGE 1 — Trim daily when 0 signals fire
──────────────────────────────────────────────────────────────────
In compile_report(), when len(qualifying_signals) == 0 (no new or 
repeat signals this run), the SMART MONEY WATCHLIST section must 
show ONLY the grouped Kalshi targets table (the "Kalshi Targets 
grouped by ticker" block). Do NOT show:
  - Per-Trader Cross-References block
  - Largest Open Positions block

When signals DO exist (len > 0), show all three blocks as today.

──────────────────────────────────────────────────────────────────
CHANGE 2 — Filter sports bets from Largest Open Positions
──────────────────────────────────────────────────────────────────
In the "Largest Open Positions" block, filter out any position 
whose title matches _is_sports_title() from 
analysis/smart_money_scan.py. Import it at the top of report.py.

Only show positions that pass _is_sports_title() == False.
If fewer than 3 positions remain after filtering, show a note:
"  (No non-sports positions large enough to display)"
instead of an empty block.

──────────────────────────────────────────────────────────────────
CHANGE 3 — Resolution countdown in header
──────────────────────────────────────────────────────────────────
Add a new logger function get_next_resolution_date() that queries 
the signals table for the earliest close_time among unresolved 
paper signals (result = '' or NULL, close_time IS NOT NULL and 
NOT ''), returns it as a date string or None if no close_times 
are stored.

In compile_report() header block, if get_next_resolution_date() 
returns a value, add one line:
  "  Next resolution: {date}  ({N} days)"
where N = days from today. If None, omit the line entirely.

──────────────────────────────────────────────────────────────────
CHANGE 4 — Upcoming resolutions block
──────────────────────────────────────────────────────────────────
Add a new logger function get_upcoming_resolutions(days=14) that 
returns paper signals (source='paper' or NULL) where:
  - result is '' or NULL (unresolved)
  - close_time is set and within the next N days
  - direction is YES or NO (not PASS)
  - ordered by close_time ASC

Add a new section to compile_report() after WHALE ACTIVITY and 
before TRACK RECORD:

====================================================================
UPCOMING RESOLUTIONS (closing within 14 days)
====================================================================

  {close_date}  {ticker:<30}  {direction}  {confidence}  {market_price:.0%}
  ...

If no upcoming resolutions, print:
  "  No picks closing within 14 days."
